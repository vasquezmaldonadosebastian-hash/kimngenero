# Informe de cambios — Fase 1 (Fundaciones)
Fecha: 2026-04-18
Repo: KimnGenero

Este informe resume lo implementado para la Fase 1 descrita en `OPTIMIZATION_PLAN.md`
(tests, contrato de errores, validación, tipado y lint).

## Backend (Express)
- Se introdujo un factory `createApp()` para separar construcción de app del arranque del servidor.
- Se incorporó un middleware global de errores con contrato:
  `{ error, code, message, details? }` (manteniendo `error` por compatibilidad).
- Se agregó validación de parámetros con Zod para rutas con `:id` y `:categoryId`.
- Se agregó logging HTTP con `pino-http` (omitido en `NODE_ENV=test`).

Archivos principales:
- `server/src/createApp.ts`
- `server/src/errors/AppError.ts`
- `server/src/middleware/error.middleware.ts`
- `server/src/middleware/validate.middleware.ts`
- `server/src/api/v1/indicators/indicators.routes.ts`
- `server/src/app.ts`

## Testing (Vitest + Supertest)
- Se añadió configuración Vitest y tests de humo para endpoints de indicadores.

Archivos:
- `vitest.config.ts`
- `server/src/api/v1/indicators/indicators.routes.test.ts`

## Frontend (API client)
- Se creó un wrapper de cliente `apiGetJson()` para centralizar manejo de errores/JSON.
- Se migró `IndicatorsContext` para consumir el wrapper.

Archivos:
- `client/src/lib/apiClient.ts`
- `client/src/contexts/IndicatorsContext.tsx`

## Tipos compartidos
- Se separó `shared/types` en:
  - `indicator-domain.ts` (dominio de indicadores + raws)
  - `common.ts` (tipos genéricos/auxiliares)
  - `indicators.ts` como barrel (re-export) para evitar breaking changes de imports.

Archivos:
- `shared/types/indicator-domain.ts`
- `shared/types/common.ts`
- `shared/types/indicators.ts`

Archivos:
